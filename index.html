SLAMS

Smart Late Arrival Management System

Chinmaya Vidyalaya Sr. Sec. School

Automates the logging, reporting, and communication of student late arrivals — replacing paper registers with a fast, transparent digital workflow.

Main Features

📝

Record Late Entry

Enter an admission number — SLAMS auto-fetches student name, class, and section. Add the reason and submit. Entry saved instantly, count updated automatically.

⚠️

Automatic Alerts

When a student's late count exceeds the configured threshold, a parent email is automatically triggered (if the toggle is enabled in Settings).

📅

Daily Report

Every day, the system auto-generates a report of all late students and sends it to the Principal's email. Optional copy to Teacher-in-Charge.

⚙️

Settings Panel

Authorised users can configure late threshold, principal email, reset mode, custom reset date, and toggle parent email notifications on or off.

Settings Panel Reference

SettingDescription
Late ThresholdNumber of allowed late entries before action triggers. E.g. set to 3 → alert fires on the 4th late.
Principal EmailReceives the auto-generated daily late arrival report every morning. Additional recipients (such as the teacher-in-charge) can be included by entering multiple email addresses separated by commas.
Reset ModeAll Time — never resets · Monthly — resets each month · Quarterly — every 3 months · Custom — from a chosen date
Custom Reset DateUsed when Reset Mode is Custom. Typically the start of a new term.
Parent Email ToggleON — emails sent after threshold  |  OFF — no emails sent

Key Features

🔐

Controlled Access

Admin panel controls which student can vote at any time. Only one student can vote at a time to prevent multiple simultaneous access.

📊

Dual Categories

Separate voting for Boys SPL and Girls SPL categories. Students can vote in both categories if enabled by admin.

🛡️

Vote Integrity

Once a vote is cast in a category, the student's status changes to 'cast' preventing duplicate votes in that category.

�️

Voter Anonymity

Votes are counted without storing the student's choice. The database tracks candidate vote totals, not who voted for whom.

�📈

Live Results

Password-protected results page shows real-time vote counts for all candidates in both Boys and Girls categories.

👁️

Admin Monitoring

Dedicated monitor page shows all registered voters and their current voting status for both categories.

⏱️

Timestamp Tracking

All votes are timestamped when cast, providing an audit trail of voting activity.

Database Structure

TableStructure & Purpose
votersid (PRIMARY KEY), regno (UNIQUE TEXT), boys_status (TEXT: 'enabled'/'cast'), girls_status (TEXT: 'enabled'/'cast'). Stores voter registration and voting status.
votesid (PRIMARY KEY), candidate (TEXT), category (TEXT: 'Boys'/'Girls'), timestamp (DATETIME). Records each vote with candidate, category, and time.
🗳️
Election Database
SQLite database with voters and votes tables. Stores registration numbers, voting status, and vote records while keeping individual ballot choices anonymous.
Student cannot voteAdmin must first enable voting for that registration number in the admin panel.
Already voted messageStudent has already cast a vote in that category. Status shows as 'cast' in database.
Results not updatingEnsure correct password is entered. Results show live data from votes table.
Multiple students votingOnly one student can have 'enabled' status at a time. Admin controls access sequentially.
Database connection errorEnsure election.db file exists and Flask app has proper database permissions.

How to Create Your Website

🧠

Step 1: Plan Your Content

Decide what the website will include: subject notes, worksheets, announcements, and links.

🤖

Step 2: Generate Website Code

Ask ChatGPT or a similar tool to create a simple website with sections for notes, worksheets, and announcements.

Copy the generated HTML code.

💻

Step 3: Create a GitHub Account

Go to github.com and sign up for a free account.

📁

Step 4: Create a Repository

Click New Repository, name it (for example: math-resources), and make it Public.

📄

Step 5: Upload Your Website

Create a file named index.html, paste your HTML code, and save it.

🌍

Step 6: Enable GitHub Pages

Go to Settings → Pages, select Branch: main and Folder: /root.

Your site will be live at https://yourusername.github.io/repository-name.

Task Automation

Working smarter, not harder with technology

Task automation means using technology to simplify repetitive, time-consuming work. The core philosophy: don't just finish work—improve the way the work is done. With tools like Excel, Python, Google Apps Script, and GenAI, any teacher can automate tasks without being a programmer.

🤖 ChatGPT ↗

Your instant assistant for learning how to automate any task or solve technical problems.

0Programming Required

The Mindset (Most Important)

🧠

Think Before Acting

When faced with any task, pause and ask: "Can this be done in a better, faster way using technology?"

✨

Work Smarter

The goal is not to work harder, but to use technology strategically to simplify your effort.

📚

Learn by Doing

Every automation task is an opportunity to learn something new and improve your skills.

🚀

Build Momentum

Initial time investment pays dividends—next time that task takes 2 minutes instead of 15.

Automation Tools & Levels

📊
Level 1: ExcelFormulas and functions. Start here. No prior knowledge needed.
🤖
Level 2: GenAI + ExcelUpload files to ChatGPT for data modification and cleaning.
Level 3: PythonFor large or complex data tasks. You don't write it—ChatGPT does.
Level 4: Google Apps ScriptAutomate workflows like sending emails or processing Google Sheets data.

What You Actually Need

Programming KnowledgeYou don't need it. Ask ChatGPT to generate code.
Good PromptingKnow how to describe your problem clearly to GenAI.
Copy-Paste SkillsCopy code from ChatGPT and paste into Excel, Python, or Google Apps.
Willingness to TryExperiment without fear. Learning happens through doing.

When You Get a Task

⏸️

Step 1: Pause

Don't start doing it manually. Take a moment and think.

🤔

Step 2: Ask Yourself

"Can this be automated? What tool can help?"

💬

Step 3: Consult ChatGPT

Describe your problem naturally. Use voice typing if it helps. Get step-by-step guidance.

🔧

Step 4: Try the Solution

Follow the guidance. Copy-paste code if needed. Execute and test.

📖

Step 5: Learn & Iterate

Understand what happened. Ask follow-up questions. Improve next time.

Upskilling means continuously learning, improving, and expanding your abilities beyond what is immediately required. It's not about doing your current work well—it's about becoming better than what your current role demands. Every task is an opportunity to learn something new. The best way to learn is by doing, experimenting, and building. When you grow, your work improves. When your work improves, your impact grows.

A Farewell Note

With Deep Gratitude

It has been five meaningful years at Chinmaya Vidyalaya, and this journey has shaped me in more ways than I can express.

When I first appeared for my interview as a primary computer science teacher, I was asked if I knew artificial intelligence, robotics, and related areas. I answered confidently—perhaps more with intent than with experience at that time. But when I entered the classroom and did not find these areas in the curriculum, I felt a strong urge to go beyond it.

That is how the Chinmaya Computing Club was born.

It was not just an idea—it was a space I created to explore what I truly loved teaching and learning. And this was possible not just because I wanted it, but because I was in an environment where such initiatives were encouraged and supported.

Over time, I explored, experimented, built systems, and worked with students beyond the classroom. I realised that I am someone who finds it difficult to stay within fixed boundaries—I learn by trying, by building, and by stepping beyond what is expected. This school gave me the space to do exactly that.

Everything I have done here—every system, every initiative—has been a learning experience for me. I did many things not because they were assigned, but because I was curious, and I wanted to see how much better things could be made.

And that curiosity has been my greatest teacher.

Today, as I move on, I do so with no hesitation and no regrets. I carry only gratitude. This transition is not driven by dissatisfaction, but by a natural progression—a desire to explore further, to grow, and to contribute in new ways, particularly in the field of artificial intelligence.

I hope to continue this journey of learning and teaching—now with older students, and in a different space—while still holding on to the same curiosity that shaped my work here.

One thing I am deeply thankful for is that I never allowed myself to become stagnant. I was constantly learning, trying, and building. That is what has kept me relevant, and that is what gives me the confidence to take this next step.

For that, I sincerely thank the school.

I am grateful to the management for the opportunities given to me, for the trust placed in me, and even for the resources provided—every system I built began from that support.

I thank my colleagues and friends for their encouragement, patience, and support throughout my journey. Every interaction, every collaboration has contributed to what I have become today.

As I leave, I take with me everything this school has shaped in me, and I hope to leave behind not just systems, but a way of seeing — that every classroom, every challenge, every idea is an opportunity waiting to be explored.

I only hope that the work continues, evolves, and becomes even better.

— Aishwarya S. Bramma
